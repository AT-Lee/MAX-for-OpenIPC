# MAX для OpenIPC
### Интеграция отправки видеороликов с камер OpenIPC в мессенджер MAX (скрипт + веб-интерфейс)

#### ! Создано при помощи GLM-5.x от Z.ai<br>на основании скриптов telegram и ntfy из состава OpenIPC

## Описание

Скрипт захватывает видеоролик с камеры OpenIPC и отправляет его в чат мессенджера [MAX](https://max.ru) через официальное [Platform API](https://dev.max.ru/docs-api). Предназначен для работы с детектором движения: при запуске `max`, он пишет сегмент видео и отправляет его в MAX. Если движение продолжается — следующие сегменты пишутся **без пауз** между ними (загрузка идёт в фоновом процессе, пока пишется следующий сегмент).

---

## Возможности

* Защита от одновременного запуска нескольких экземпляров `max`. При попытке повторного запуска во время записи — автоматическое продление (extend) без дублирования кадров.

* Захват и загрузка работают параллельно через файловую очередь. Пауза между сегментами — **менее 1 секунды**.

* Прерывание записи, если свободное место в `/tmp` падает ниже заданного процента (по умолчанию 10%).

* Настройка всех параметров через стандартную веб-морду OpenIPC (Extensions -> MAX).

## Установка

##### 1. Установка файлов

Скопируйте файлы на камеру (например, через WinSCP или `scp`):

```
max       ->  /usr/sbin/max
max.conf  ->  /etc/webui/max.conf
ext-max.cgi -> /var/www/cgi-bin/ext-max.cgi
```

##### 2. Настройка прав

Зайдите на камеру по SSH и выполните:

```sh
chmod +x /usr/sbin/max
chmod +x /var/www/cgi-bin/ext-max.cgi
```

##### 3. Добавление в меню (опционально)

Для добавления пункта меню во вкладку "Extensions" отредактировав файл /var/www/cgi-bin/p/header.cgi добавив после 76 строки новую строку:

```html
<li><a class="dropdown-item" href="ext-max.cgi">MAX</a></li>
```

## Конфигурация

Все параметры хранятся в `/etc/webui/max.conf`:

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `max_enabled` | `true` | Главный выключатель. `false` — отключает без удаления файлов. |
| `max_token` | — | **Обязательно.** Токен бота от MAX BotFather. |
| `max_chat_id` | — | **Обязательно.** ID целевого чата (отрицательный для групп). |
| `max_caption` | `%hostname, %datetime` | Шаблон подписи. Поддерживает `%hostname`, `%datetime`, `%soctemp`. Индекс сегмента `(#N)` добавляется автоматически. |
| `max_video_duration` | `10` | Длительность одного сегмента в секундах. Диапазон 1–30. Длинное движение = больше сегментов, а не более длинные сегменты. |
| `max_min_free_pct` | `10` | Прервать запись, если свободного места в `/tmp` меньше этого процента. Защищает флэш-память камеры. |
| `max_proxy` | (нет) | `true` — использовать SOCKS5 из `/etc/webui/proxy.conf`. |
| `max_interval` | `15` | Интервал в минутах для cron-отправки (только если `max_crontab=true`). |
| `max_crontab` | `true` | Добавить `*/N * * * * /usr/sbin/max` в `/etc/crontabs/root`. |

## Использование

### По детектору движения

Добавьте /usr/sbin/motion.sh:

```sh
#!/bin/sh
/usr/sbin/max
```

Теперь при каждом срабатывании детектора движения камера запускает `max`. Если движение продолжается и детектор срабатывает повторно — повторный запуск **продлевает** уже работающий экземпляр (через флаг `/tmp/max.extend`), сегменты не теряются.

### Расписание (cron)

В веб-интерфейсе включите **«Add to crontab»** и выберите интервал.

### Webhook (внешний триггер)

```sh
curl 'http://<логин>:<пароль>@<ip_камеры>/cgi-bin/ext-max.cgi?send=test'
# → OK    (захват и отправка прошли успешно)
# → FAIL  (что-то пошло не так, смотрите вывод /usr/sbin/max через SSH)
```

Это тот же endpoint, что вызывает кнопка «Send test video» в веб-интерфейсе.

### Веб-интерфейс

Откройте в браузере `http://<ip_камеры>/cgi-bin/ext-max.cgi`. На странице:

- Жёлтое предупреждение, если `hls` не включён в `majestic.yaml`.
- Форма со всеми параметрами `max.conf`.
- Кнопка **«Send test video»**.
- URL webhook'а (кликабельный, копируется по клику).

## Как это работает

### Архитектура

```
┌──────────────── ГЛАВНЫЙ ПРОЦЕСС ───────────────┐
│                                                │
│  цикл:                                         │
│    1. проверить df /tmp (прервать если < %)    │
│    2. curl /video.mp4?duration=N → /tmp/seg    │
│    3. добавить "путь|подпись" в очередь        │
│    4. если /tmp/max.extend существует:         │
│         удалить, повторить цикл                │
│       иначе:                                   │
│         выйти, сигнал DONE воркеру             │
│                                                │
└──────────────────────┬─────────────────────────┘
                       │ файл очереди
                       ▼
┌──────────────── ВОРКЕР ЗАГРУЗКИ (фон) ─────────┐
│                                                │
│  цикл:                                         │
│    1. прочитать первую строку из очереди       │
│    2. POST /uploads?type=video → {url, token}  │
│    3. POST multipart с файлом → url            │
│    4. POST /messages с видео-вложением         │
│    5. retry при attachment.not.ready           │
│    6. удалить строку из очереди, повтор        │
│                                                │
└────────────────────────────────────────────────┘
                       ▲
                       │ touch /tmp/max.extend
┌───────────────────────┴────────────────────────┐
│  ВТОРОЙ ВЫЗОВ max (пока первый работает)       │
│    → ставит флаг extend, мгновенно выходит     │
└────────────────────────────────────────────────┘
```
